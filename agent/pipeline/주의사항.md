# 주의사항

## Elasticsearch 라이선스 제한

### RRF (Reciprocal Rank Fusion) 사용 불가

- **현상**: Hybrid 검색에서 `retriever.rrf` 사용 시 403 에러 발생
  ```
  AuthorizationException(403, 'security_exception',
  'current license is non-compliant for [Reciprocal Rank Fusion (RRF)]')
  ```
- **원인**: 현재 ES 클러스터(`elasticsearch-edu.didim365.app`)의 라이선스가 Basic/Standard이며, RRF는 **Platinum 이상** 라이선스에서만 사용 가능
- **대응**: `query` + `knn` 동시 전송 방식으로 Hybrid 검색 구현 (ES가 내부적으로 score를 합산)
- **참고**: RRF는 순위 기반 합산이라 스코어 스케일이 다른 BM25/Vector 결합에 더 적합하나, 현재 방식도 실용적으로 충분함

### Elastic Rerank 모델 사용 불가

- **현상**: ES 내장 `text_similarity_reranker` retriever 및 `.rerank-v1-elasticsearch` inference endpoint 사용 불가
- **원인**: Elastic Rerank 모델은 **Elastic Cloud 유료 플랜** 또는 **Platinum 이상** 라이선스 필요
- **대응**: Cohere Rerank v3.5 API를 외부 호출하여 ReRanker 기능 구현 (`search.py --rerank`)
