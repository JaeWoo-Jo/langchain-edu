## 실행 방법

### 전체 워크플로우 (순서대로)

```bash
cd agent/

# 1단계: Opik 평가 실행 (100개 데이터셋)
#   → 에이전트를 100번 호출하고 4개 메트릭으로 채점
#   → 결과가 Opik 서버에 자동 저장됨
uv run python -m tests.test_opik_eval

# 2단계: DeepEval 평가 실행 (10개 대표 케이스)
#   → TaskCompletion, ToolCorrectness, GEval로 채점
uv run deepeval test run tests/test_deepeval.py

# 3단계: 리포트 자동 생성
#   → Opik에서 데이터 ��져와서 분석
#   → docs/week2-performance-report.md 에 저장
uv run python tests/generate_report.py
```

### 리포트만 재생성

이미 평가를 돌린 후 리포트만 다시 만들고 싶을 때:

```bash
uv run python tests/generate_report.py
```

## 리포트 ���조

생성되는 리포트(`docs/week2-performance-report.md`)의 섹션:

1. **개요** — 평가 체계 요약
2. **트레이스 요약** — 성공/에러율, 응답 시간, 도구 호출 분포
3. **메트릭별 성능** — 각 메트릭의 평균/최저/최고/미달 수
4. **취약 지점 분석** — 임계값 미달 메트릭의 상세 분석
5. **개선 우선순위** — 심각도 순 정렬된 개선 과제 표
6. **다음 단계 제안** — 3주차와 연계한 후속 작업

## 3주차와의 연결

커리큘럼의 교육생 조언:
> 2주차에 배운 평가 지표를 활용해, 3주차 하이브리드 검색 도입 후
> Task Completion 점수가 얼마나 개선되었는지 숫자로 증명해 보세요.

따라서 이 리포트의 점수는 **3주차 개선 전 베이스라인**이 된다.
3주차에 하이브리드 검색(RRF)을 도입한 후 동일 평가를 재실행하여 점수 변화를 비교하면 된다.
