### 3. 주요 개선 사항

#### 3-1. 프롬프트 최적화

- 문제: 도구 5~6회 반복 호출, 불필요한 차트 생성
- 해결: 호출 제한 3회, 도구 설명 명확화, 계획 규칙 추가
- 효과: "감자 얼마야?" 도구 호출 5회→1회, 응답 ~6초→~2초

#### 3-2. 검색 품질 개선

- 문제: _score=0 (date 정렬), multi_match 노이즈, 중복
- 해결: _score 기반 정렬, minimum_should_match 75%, BM25+kNN 하이브리드, item_name+kind_name 중복 제거
- 효과: 검색 점수 0→1.06, 감자 중복 4건→2건, 노이즈 제거

#### 3-3. Plan-Execute-Reflect 딥에이전트

- 문제: 단순 ReAct 패턴으로 복합 질문 처리 한계
- 해결: Planner→Executor→Reflector→Synthesizer 4단계 StateGraph
- 효과: 복합 질문 체계적 처리, replan 루프로 자가 교정

#### 3-4. 평가 체계 고도화

- 문제: 단일 GEval로 모든 케이스 채점 → 부당 감점
- 해결: 카테고리별 GEval 분리(가격/RAG/일반), 데이터셋 100→115건, LLM-as-a-judge
- 효과: "안녕!" GEval 0.26→0.81 등 공정한 평가
