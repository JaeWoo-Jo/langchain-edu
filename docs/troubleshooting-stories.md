# 어려웠던 점 및 극복 방법

에이전트를 다양한 시나리오로 테스트하며 발견한 문제와 해결 과정을 정리했습니다.

## 1. 복수 품목 비교 시 한쪽 가격 누락

**증상**: "쌀과 콩 가격 비교" 질문 시 첫 응답에서 쌀 가격을 못 찾고 콩만 반환. 두 번째 질문에서야 쌀이 나옴.

**재현 방법**: `쌀과 콩 가격 비교`

**Before**:
> ㅋㅋ 쌀 가격 정보는 없지만, 콩 가격은 500g에 오늘 5116원이야. 쌀 가격 알고 싶으면 따로 알려줘~

**원인 분석**: 3가지 원인이 겹침
1. **PLANNER_PROMPT**의 "같은 도구를 2번 이상 호출하지 않는다" 규칙이 search_price를 품목별로 나눠 호출하는 것을 차단
2. **Reflector**의 `step_summary`가 `recent_results[-1]`(마지막 결과)만 저장 → 쌀 결과 유실
3. **Synthesizer**가 `step_results`만 보고 답변 → 유실된 데이터로 "쌀 정보 없음" 판단

**수정 내용**:

| 파일 | 변경 |
|------|------|
| `prompts.py` PLANNER | "같은 도구 2번 호출 금지" 제거, 최대 단계 3→5, 여러 품목은 별도 단계로 분리 명시 |
| `deep_agent.py` reflector | `recent_results[-1]` → `" \| ".join(recent_results)` (전체 결과 보존) |
| `deep_agent.py` synthesizer | `step_results` 대신 실제 `ToolMessage` 내용을 직접 수집 |

**After**:
> ㅋㅋ 쌀이랑 콩 가격 비교해줄게~
> - 쌀 20kg: 오늘 62,038원, 1주 전 대비 250원 내림
> - 쌀 10kg: 오늘 35,589원, 1주 전 대비 1,111원 내림
> - 콩 (흰 콩, 500g): 오늘 5,116원, 1주 전 대비 125원 내림
>
> 단가로 보면 쌀 20kg는 1kg당 약 3,102원, 콩 500g은 5,116원이니까 1kg당 약 10,232원. 콩이 쌀보다 대략 3배 이상 비싸 ㅋㅋ

**교훈**: 상태 전달 체인(Reflector → step_results → Synthesizer)에서 데이터가 유실되면 최종 답변이 틀려진다. 요약 단계마다 정보가 손실되지 않는지 end-to-end로 확인해야 한다.

## 2. 영어/이모지 입력 시 품목 검색 실패

**증상**: 영어로 "How much is rice?" 질문 시 쌀 가격을 못 찾음. "영어로 쌀 가격 알려줘"에도 영어로 답변하며 가격 누락.

**재현 방법**: `How much is rice?` / `영어로 쌀 가격 알려줘`

**Before**:
> 쌀 가격 정보는 현재 데이터에 없어 ㅠㅠ 다른 궁금한 가격 있으면 알려줘!ㅋㅋ

> Sorry, there is no current price data for rice available.

**원인 분석**:
- Executor가 `search_price("rice")`로 도구를 호출 → Elasticsearch의 한글 인덱스에서 "rice" 매칭 실패
- Synthesizer에 페르소나 유지 규칙이 없어 영어 요청 시 영어로 답변

**수정 내용**:

| 파일 | 변경 |
|------|------|
| `prompts.py` EXECUTOR | "도구에 전달하는 품목명은 반드시 한글로 변환" 규칙 추가 (rice→쌀, potato→감자) |
| `prompts.py` SYNTHESIZER | "사용자가 영어로 요청해도 반말+한글로 답변" 규칙 추가 |

**After**:
> 쌀 가격은 20kg에 62,038원이고, 10kg에 36,700원이야. 20kg 쌀이 1주 전보다 250원 싸졌고, 10kg 쌀 가격은 1주 전이랑 같아. ㅋㅋ 참고해~

> 쌀 20kg는 오늘 62,038원, 쌀 10kg는 36,700원이야 ㅋㅋ 필요한 무게에 따라 선택하면 돼!

**교훈**: 도구의 입력 인터페이스(한글 품목명)와 사용자 입력(다국어)의 간극을 Executor 프롬프트에서 번역 레이어로 메워야 한다.

## 3. 카테고리 질문 시 검색 실패 ("가장 싼 채소 추천해줘")

**증상**: 채소 가격 데이터 54건이 있는데도 "채소 가격 정보가 없어"라고 답변.

**재현 방법**: `가장 싼 채소로 요리 추천해주고 영양성분도 알려줘`

**Before**:
> 채소 가격 정보가 없어서 정확한 최저가 채소는 알려줄 수 없어 ㅠㅠ 대신 싸고 쉽게 구할 수 있는 채소로 할 수 있는 간단 요리를 추천할게!

**원인 분석**:
- Planner가 `search_price("채소")`로 계획 → "채소"라는 품목은 ES에 없음
- 개별 채소명(배추, 무, 양파 등)으로 검색해야 매칭됨
- Planner에 카테고리→대표 품목 분해 규칙이 없음

**수정 내용**:

| 파일 | 변경 |
|------|------|
| `prompts.py` PLANNER | 카테고리 질문 시 대표 품목 3~4개를 선정하여 개별 검색하는 규칙 추가 |

추가된 규칙:
```
"싼 채소 추천" → 1단계: search_price("배추"), 2단계: search_price("무"),
                 3단계: search_price("양파"), 4단계: 결과 종합
```

**After**:
> 가장 저렴한 채소는 무야! 현재 1개당 약 1,800원대에 구매할 수 있어.
>
> 무 요리 추천: 무국, 무생채, 무조림
> 무의 영양성분: 비타민 C, 식이섬유, 칼슘, 칼륨

**교훈**: LLM은 추상적 카테고리("채소")를 구체적 검색어("배추", "무")로 자동 분해하지 못한다. 프롬프트에서 분해 전략을 명시적으로 가르쳐야 한다.

## 4. 가격 데이터 부족 (식량작물 12건만 수집)

**증상**: 당근, 양파, 사과, 돼지고기 등 일반적인 품목의 가격을 전혀 찾지 못함.

**재현 방법**: `당근이랑 양파 가격 비교해줘`

**Before**:
> 당근이랑 양파 가격 정보가 현재 데이터에 없어서 비교해 줄 수가 없어 ㅠㅠ

**원인 분석**:
- `collect_prices.py` 스크립트가 KAMIS API의 **식량작물(100) 카테고리만** 수집
- 채소류(200), 과일류(400), 축산물(500), 수산물(600) 등 5개 카테고리가 누락
- 결과: 12건(쌀, 감자, 고구마 등 7종)만 적재

**수정 내용**:

| 파일 | 변경 |
|------|------|
| `scripts/collect_prices.py` | 6개 카테고리 전체를 순회하며 수집하도록 변경 |

```python
# Before: 카테고리 없이 호출 (기본값 식량작물만)
data = fetch_prices(args.date)

# After: 6개 카테고리 순회
CATEGORIES = {"100": "식량작물", "200": "채소류", "300": "특용작물",
              "400": "과일류", "500": "축산물", "600": "수산물"}
for code, name in CATEGORIES.items():
    data = fetch_prices(args.date, category_code=code)
```

**After** (12건 → 175건 적재):
> 당근은 무세척 기준으로 1kg에 2,485원~3,314원 사이고, 양파는 1kg에 1,894원~1,920원이야. 양파가 당근보다 대체로 1,000원 이상 저렴하지 ㅋㅋ

| 카테고리 | 건수 | 주요 품목 |
|---------|------|----------|
| 식량작물 | 12건 | 쌀, 감자, 고구마, 콩 등 |
| 채소류 | 54건 | 당근, 양파, 배추, 토마토 등 |
| 특용작물 | 11건 | 참깨, 버섯, 호두 등 |
| 과일류 | 23건 | 사과, 바나나, 딸기 등 |
| 축산물 | 29건 | 소, 돼지, 닭, 계란 등 |
| 수산물 | 46건 | 고등어, 갈치, 새우 등 |

**교훈**: 에이전트 성능의 상당 부분은 프롬프트나 모델이 아니라 **데이터 커버리지**에 달려있다. 에이전트가 "못 찾겠다"고 할 때 모델 탓 전에 데이터부터 확인해야 한다.

## 5. 존재하지 않는 품목에 대한 가격 추측 (할루시네이션)

**증상**: 검색 결과가 없는 품목에 대해 에이전트가 가격을 지어냄.

**재현 방법**: `콩과 당근 가격 비교` (당근 데이터 없던 시점)

**Before**:
> 흰 콩 500g에 5,116원이라고 해! 당근 가격 정보는 이번에 못 찾았어 ㅠㅠ **대체로 당근은 1kg에 1,500~2,000원 정도 하니까**, 콩이 당근보다 훨씬 비싼 편이야 ㅋㅋ

**원인 분석**:
- Synthesizer 프롬프트에 "추측 금지" 규칙이 없음
- LLM이 학습 데이터의 일반 상식으로 가격을 추정하여 답변
- 프롬프트의 "추측으로 가격을 말하지 마" 규칙은 PRICE_SYSTEM_PROMPT에만 있고 SYNTHESIZER_PROMPT에는 없었음

**수정 내용**:

| 파일 | 변경 |
|------|------|
| `prompts.py` SYNTHESIZER | "도구 결과에 없는 품목의 가격을 절대 추측하지 마" 규칙 추가 |
| `prompts.py` SYNTHESIZER | "보통 ~원 정도" 같은 추정 표현 사용 금지 명시 |

**After**:
> 흰 콩(국산, 500g)은 오늘 5,116원인데, 당근 가격 정보는 없어 ㅠㅠ 그래서 둘 직접 비교는 힘들어 ㅋㅋ

**교훈**: LLM의 할루시네이션은 "하지 마"보다 "어떻게 대응해"를 알려줘야 막을 수 있다. "추측하지 마" + "없으면 솔직히 없다고 말해"를 함께 넣어야 효과적이다.
